# Contributing

Спасибо за интерес к проекту Skin Cancer AI.

Этот репозиторий объединяет несколько частей системы:

- `frontend/` — пользовательский интерфейс на React/Vite
- `backend/` — FastAPI, auth, API v1, интеграции
- `ml/` — ML-сервис инференса
- `nginx/` — reverse proxy

## Перед началом

Перед любыми существенными изменениями рекомендуется:

1. Открыть issue с кратким описанием проблемы или идеи.
2. Уточнить, меняется ли публичный API, поведение ML-пайплайна или внешняя интеграция.
3. Для крупных изменений сначала согласовать подход, а потом готовить PR.

## Локальный запуск

Базовый способ запуска для разработки и проверки — через Docker Compose:

```bash
cp .env.example .env
docker compose up --build -d
```

Дальше:

- frontend: `http://localhost:90`
- backend health: `http://localhost:90/backend/health`

Подробности по окружению и переменным:

- деплой и env: [docs/DEPLOY.md](docs/DEPLOY.md)
- HTTP API: [docs/API.md](docs/API.md)
- пользовательский сценарий: [docs/USER_GUIDE.md](docs/USER_GUIDE.md)

## Что важно учитывать

### Медицинский и исследовательский контекст

Проект относится к исследовательским инструментам анализа изображений кожи. При изменениях не упрощайте и не удаляйте:

- дисклеймеры
- ограничения модели
- предупреждения о не-медицинском назначении

См. также: [docs/MODEL_LIMITATIONS.md](docs/MODEL_LIMITATIONS.md) и [docs/DISCLAIMER.md](docs/DISCLAIMER.md).

### Внешние контракты

Особенно аккуратно относитесь к изменениям в следующих зонах:

- `docs/API.md` и публичный API v1
- auth и API-ключи
- интеграция с внешним сервисом `img2txt`
- callback `POST /internal/description-results/{job_id}`

Если меняется контракт запроса/ответа, обновите документацию в том же PR.

## Pull Request Checklist

Перед открытием PR проверьте:

1. Изменение решает одну понятную задачу.
2. README и/или docs обновлены, если меняется поведение системы.
3. Если менялся API-контракт, это отражено в `docs/API.md`.
4. Если менялась интеграция с `img2txt`, это отражено в `docs/API.md` и `docs/DEPLOY.md`.
5. Если менялась ML-логика или ожидания от модели, обновлены ограничения/оговорки в `docs/MODEL_LIMITATIONS.md`.
6. В PR-описании указаны риски, ограничения и способ проверки.

## Стиль изменений

- Старайтесь делать небольшие, сфокусированные PR.
- Не смешивайте refactor, feature и документацию без необходимости.
- Если поведение спорное или неоднозначное, опишите мотивацию прямо в PR.

## Security

Не публикуйте:

- реальные ключи и токены
- `.env`
- приватные URL и секреты интеграций
- персональные или медицинские данные пользователей

Если вы нашли уязвимость, не создавайте публичный issue. См. [SECURITY.md](SECURITY.md).
