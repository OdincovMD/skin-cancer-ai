# Model Limitations

## Назначение

Skin Cancer AI — исследовательский инструмент для анализа дерматоскопических изображений по методу Киттлера.

Он не является медицинским изделием и не должен использоваться как единственный источник клинического решения.

См. также: [docs/DISCLAIMER.md](DISCLAIMER.md).

## Что делает система

Система:

- принимает дерматоскопическое изображение
- строит маску области интереса
- извлекает набор визуальных признаков
- формирует дерево решений и итоговый класс
- при включённой внешней интеграции может дополнительно генерировать текстовое описание и признаки через `img2txt`

## Ключевые ограничения

### 1. Это не клиническая валидация

Наличие модели, дерева решений и текстового описания не означает клиническую пригодность для самостоятельной диагностики.

Результат нельзя интерпретировать как медицинское заключение.

### 2. Качество результата зависит от входного изображения

Результат может быть ненадёжным, если:

- изображение размыто
- низкое разрешение
- плохое освещение
- сильные артефакты, тени, блики
- объект снят не в дерматоскопическом формате
- область поражения частично закрыта

### 3. Ограниченность обучающих данных и смещений

Поведение модели определяется доступными данными, на которых были обучены её компоненты. Это может создавать:

- bias по типам кожи
- bias по качеству и способу съёмки
- слабую обобщающую способность на редкие случаи
- нестабильность на изображениях вне ожидаемого домена

## 4. Ограничения интерпретируемости

Хотя проект использует признаки и дерево решений, интерпретируемость здесь ограниченная:

- наличие дерева решений не гарантирует клиническую объяснимость
- промежуточные признаки тоже могут быть предсказаны с ошибкой
- ошибка раннего шага может исказить весь последующий путь

## 5. Ограничения внешнего текстового описания

Если включена интеграция с `img2txt`:

- текстовое описание является дополнительным слоем интерпретации, а не источником истины
- описание может отставать по времени от основной классификации
- признаки и текст могут зависеть от версии внешнего сервиса
- при изменении внешнего API или модели описание может поменяться без изменения основной ML-части этого репозитория

## 6. Ограничения эксплуатации

Проект не следует использовать:

- для автоматического triage без участия специалиста
- как единственный фильтр злокачественности
- в сценариях, где ошибка может напрямую повлиять на здоровье пациента без независимой врачебной проверки

## Рекомендации по ответственному использованию

- Использовать только как исследовательский или вспомогательный инструмент.
- Сопоставлять результат с клиническим контекстом.
- Показывать пользователю дисклеймер.
- Не скрывать неопределённость результата.
- При публикации демо и статей описывать ограничения модели явно.
